## 资源目录

```text
assets/screenshots/  README 和演示材料使用的页面截图
dev/                 开发过程、测试规则、交接和维护说明
openapi/             API 契约文件
```

## 维护原则

- 新增或调整页面 / 链路后，同步更新 `前后端联调文档.md`。
- API 入参、响应、错误码或状态口径变化时，同步更新 `API设计.md`、`API风格规范.md` 和 OpenAPI 契约。
- 前端视觉或页面表达变化时，优先校准 `前端产品与页面设计准则.md` 与根目录 `DESIGN.md`。
- 历史开发文档只作参考，当前能力以实际代码、核心产品/API 文档和用户明确需求为准。
