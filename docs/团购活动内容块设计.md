# 团购活动内容块设计

## 1. 定位

团购活动正文由一组有序的结构化内容块构成。内容块支持团长表达开团理由、活动亮点、实拍展示、发货说明等结构化信息。

**核心原则**：团购正文属于 `group_buys` 活动本身，不属于商品。用户进入团购详情看到的是活动内容，不是商品详情。

## 2. 实现批次

- P1 Batch 07：后端支持 `content_blocks` 字段，提供结构校验、JSON 序列化/反序列化、安全限制和 API 响应。
- 后续批次：前端编辑器、渲染组件、内容块模板。

## 3. 存储

- 字段：`group_buys.content_blocks`，类型 `TEXT`（存 JSON 字符串）。
- 实体层用 `String` 存储 JSON，Service 层通过 `ContentValidationUtil` 做序列化/反序列化。
- 最多 20 块，保持原始顺序。
- 空字段在 API 响应中返回空数组 `[]`。

## 4. 内容块类型

| 类型 | 用途 | 支持字段 |
|---|---|---|
| `paragraph` | 活动正文段落 | `text`（最多 1000 字） |
| `section` | 小标题 + 正文 | `title`（最多 40 字），`text`（最多 1000 字） |
| `image` | 活动图片、实拍、产地展示 | `url`（必填，最多 512 字符，仅 http/https），`caption`（最多 80 字） |
| `list` | 卖点、注意事项、适用场景 | `items`（最少 1 项，最多 10 项，单项最多 80 字） |
| `deliveryNote` | 发货、截单和履约说明 | `text`（最多 1000 字） |

## 5. 校验规则

执行入口：`ContentValidationUtil.validateContentBlocks()`

- `type` 必须来自白名单 `[paragraph, section, image, list, deliveryNote]`。
- 所有文本字段禁止 HTML 标签、`<script>`、`<iframe>`、`<style>`、事件属性、`javascript:`、`data:`。
- 数量、长度限制见上表。
- 非法类型或内容返回 `VALIDATION_ERROR`。

## 6. API 格式

请求（`POST /api/v1/my/store/group-buys` / `PATCH /api/v1/my/store/group-buys/{id}`）：

```json
{
  "contentBlocks": [
    {
      "type": "section",
      "title": "为什么开这个团",
      "text": "这批蜜桃来自固定合作果园，成熟度更稳定。"
    },
    {
      "type": "image",
      "url": "https://example.com/orchard.png",
      "caption": "果园实拍"
    },
    {
      "type": "list",
      "items": ["产地直发", "坏果包赔"]
    }
  ]
}
```

PATCH 语义：传入数组时整体替换；传 `[]` 表示清空；不传表示不改。

响应：内容块在 `groupBuy.contentBlocks` 字段中返回，结构与请求一致。

## 7. 旧数据降级

- 旧团购没有 `content_blocks` 时，API 返回 `[]`。
- 前端可使用 `groupBuy.introduction` 按段落降级渲染。

## 8. 安全限制

- 不接受任意 HTML。
- 不接受 `<script>`、`<iframe>`、`<style>`、事件属性（如 `onclick`）、内联样式。
- 不接受 `javascript:`、`data:` 等非 HTTP 图片地址。
- 不引入 HTML 富文本编辑器和渲染器。

## 9. 关系说明

```
GroupBuy 1 ── N ContentBlock   (content_blocks JSON)
GroupBuy 1 ── N GroupBuyItem   (group_buy_items)
GroupBuyItem N ── 1 Product     (products)

GroupBuy.content_blocks = 团购活动正文（本文件定义）
Product.description + Product.detailImageUrls = 商品图文说明
```

团购活动正文和商品图文说明是独立的数据源。同一个商品被复用到不同团购时，商品描述变更不影响团购活动正文。
