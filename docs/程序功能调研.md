## 1. 产品定位

快团团可以抽象为一个基于微信生态的 **用户即团长型私域社群团购小程序**。

它不是传统货架电商，也不是严格区分“买家端”和“卖家端”的商家后台系统，而是围绕“用户可随时开店、团长开团、团员跟团、微信社群传播、订单履约、复购推荐”展开。截图显示，用户在“我的 / 个人中心”里天然拥有店铺、团员、社群、一键开团、创建店铺、会员卡、帮卖分销等入口，说明产品默认把每个用户都视为潜在团长。

核心交易链路是：

```text
团长发布团购 → 商品进入信息流/团长主页 → 用户浏览商品 → 领取优惠/跟团购买 → 微信支付 → 团长发货/履约 → 用户查看订单/售后 → 分享传播/复购
```

核心经营激活链路是：

```text
普通用户进入小程序 → 创建店铺/拥有团长主页 → 完善店铺资料 → 邀请团员订阅 → 发布团购 → 推送开团通知 → 微信分享传播 → 订单成交 → 会员成长/复购/帮卖分销
```

核心特点：

- **用户可激活团长身份**：普通用户与团长没有强隔离，每个用户都可以低门槛开店、开团、邀请订阅和参与帮卖。
- **团长与店铺绑定但不完全等同**：团长是前台信任身份，店铺是后台经营容器。买家看到“团长”，经营者配置“店铺”。
- **团购是店铺下的一次销售活动**：一次团购由“介绍、商品、设置”三部分组成，不等同于店铺本身。
- **商品既可以作为店铺商品沉淀，也可以被某次团购引用售卖**：商品有名称、图片、描述、规格、价格、库存等字段。
- 商品以团购活动形式售卖，弱化传统店铺货架，强化“团购内容 + 团长信用 + 社群传播”。
- 商品详情、团长主页、分享海报、订阅邀请卡是核心转化场景。
- 微信支付、微信分享、公众号/服务通知构成基础闭环。
- 会员卡、成长值、积分商城、订阅提醒用于沉淀团长私域用户和提升复购。
- 帮卖分销被前置到店铺创建和团购设置中，是快团团区别于普通团购工具的重要能力。

## 2. 核心角色与对象模型

快团团的角色不应简单拆成“买家”和“卖家”。从截图看，平台把用户默认设计成“潜在团长”，用户可以先作为团员购买，也可以随时开店、开团、邀请订阅、参与帮卖。但进一步结合“创建店铺”“发布团购”“帮卖分销设置”等截图后，不能再把“用户、团长、店铺、团购、商品”完全混为一体，而应采用分层模型。

### 2.1 分层关系

```text
用户账号
  ↓ 激活经营能力
团长身份
  ↓ 创建或拥有经营载体
店铺 / 小店
  ↓ 发布一次次销售活动
团购活动
  ↓ 承载具体售卖对象
商品 / SKU
```

最核心的判断是：

> **团长是前台信任身份，店铺是后台经营容器，团购是一次销售活动，商品是被售卖的具体对象。**

买家端通常看到“团长”：头像、昵称、成员数、关注数、金牌团长、冠军团长、客服、订阅、团员晒单等。经营端通常配置“店铺”：店铺名称、店铺头像、店铺简介、店铺物流、帮卖分销、店铺商品等。

### 2.2 角色 / 对象定义

| 层级 | 角色 / 对象 | 定义 | 主要行为 / 字段 |
|---|---|---|---|
| 账号层 | 基础用户 | 所有进入小程序的人，默认拥有购买和经营入口 | 浏览商品、跟团购买、查看订单、收藏浏览、进入一键开团/创建店铺 |
| 关系层 | 团员 | 用户与某个团长产生订阅、购买、会员或积分关系后的身份 | 订阅团长、领取优惠券、跟团购买、成为会员、积累成长值、查看会员卡 |
| 身份层 | 团长 | 用户激活经营能力后的身份，是买家视角下的信任主体 | 编辑主页、邀请订阅、发布团购、承接咨询、沉淀成员/关注 |
| 经营层 | 店铺 / 小店 | 团长的经营载体，承载配置和资产 | 店铺名称、头像、简介、物流、帮卖分销、店铺商品、主页展示 |
| 活动层 | 团购活动 | 店铺发布的一次销售活动，不等同于店铺 | 团购介绍、团购商品、团购设置、开团通知、团购时间、发货时间 |
| 商品层 | 商品 / SKU | 被售卖的具体货品或服务 | 名称、描述、图片、规格、价格、库存、更多商品设置 |
| 分销层 | 供货团长 | 提供商品并设置帮卖规则的一方 | 设置店铺商品为可帮卖商品、设置佣金、承担交易服务费、供货/履约 |
| 分销层 | 帮卖团长 | 帮其他团长销售商品并获得佣金的人 | 分享或复制团购、卖出商品、获得固定佣金或差价收益 |
| 服务层 | 团长 / 店铺客服 | 绑定当前团长、店铺或团购的交易咨询入口 | 商品咨询、发货时间、规格问题、订单备注、售后沟通 |
| 平台层 | 官方客服 / 平台运营 | 平台规则和兜底服务角色 | 账号问题、支付异常、投诉举报、纠纷处理、内容风控、规则管理 |

### 2.3 团长与店铺的区别

| 对比项 | 团长 | 店铺 / 小店 |
|---|---|---|
| 本质 | 身份、人设、信任主体 | 经营容器、配置资产 |
| 面向对象 | 主要面向买家/团员 | 主要面向经营者/系统配置 |
| 前台展示 | 团长头像、昵称、成员数、关注数、跟团数、好评、标签 | 店铺主页、商品列表、会员权益、积分商城、帮卖介绍 |
| 后台配置 | 编辑主页、邀请订阅、开团通知 | 店铺名称、店铺头像、店铺简介、店铺物流、帮卖分销 |
| 业务价值 | 建立信任、沉淀私域关系、促进复购 | 承载商品、物流、分销、主页、订单履约能力 |

因此，快团团不是：

```text
用户 = 团长 = 店铺 = 团购
```

而更接近：

```text
一个用户可以成为团长；
一个团长拥有或创建一个店铺；
一个店铺可以发布多个团购；
一个团购可以包含一个或多个商品；
商品可以沉淀为店铺商品，也可以被具体团购引用。
```

### 2.4 客服体系推断

从截图看，“客服”按钮出现在团长主页、商品详情、团购详情等交易场景中；个人中心又存在“官方客服”入口。因此客服体系更可能是双层模型：

| 客服类型 | 归属 | 主要负责 |
|---|---|---|
| 团长 / 店铺客服 | 当前团长、当前店铺或当前团购 | 商品咨询、规格说明、发货时间、地址备注、售后沟通、履约问题 |
| 官方客服 | 快团团平台 | 账号问题、支付异常、平台规则、投诉举报、纠纷处理、系统问题 |

更准确的服务链路是：

```text
买家购买某团长商品
    ↓
优先联系当前团长 / 店铺客服
    ↓
若涉及平台规则、支付异常或纠纷
    ↓
转向官方客服兜底处理
```

## 4. 底部 Tab 导航

截图中底部固定导航包含 5 个一级入口：

| Tab | 页面定位 | 主要功能 |
|---|---|---|
| 首页 | 团购商品发现页 | 商品推荐、分类筛选、搜索、领券、订阅、进入商品详情 |
| 订单 | 用户订单中心 | 查看全部、待支付、发货、售后订单，搜索商品订单，空态推荐关注团长商品 |
| 一键开团 | 用户转团长的经营入口 | 新手/有经验分流、选择开团类型、查看教程、发布团购、使用商品库 |
| 消息 | 消息通知中心 | 活动通知、订单通知、物流通知、客服消息、公众号提醒、打开提醒引导 |
| 个人中心 | 用户资产与团长经营混合中心 | 钱包、地址、优惠券、会员卡、店铺、社群、创建店铺、订阅邀请、客服、设置、收藏浏览 |

其中“一键开团”和“个人中心”的设计共同说明：快团团没有把团长端隐藏在单独后台，而是把团长经营能力直接放在普通用户路径中，降低开店和开团门槛。

## 5. 买家端页面与功能

### 5.1 首页 / 团购信息流页

首页是买家发现团购的主要入口。

**页面元素：**

- 顶部 Logo / 搜索框。
- 公众号关注提醒：提示关注公众号以接收活动、订单、物流通知。
- 一级频道：全部、视频、最新。
- 商品分类：综合、生鲜、食品、服饰、美妆、百货等。
- 领券入口。
- 商品卡片流。
- 购物车浮动按钮。
- 底部 Tab 导航。

**商品卡片包含：**

- 团长头像 / 团长名称 / 团长标签。
- 回复快、回头客多、金牌团长等信任标签。
- 商品标题。
- 商品图片或视频。
- 价格。
- 包邮、限时秒杀、多少人在抢等营销标签。
- 跟团人数、浏览人数。
- 分享按钮。
- 去跟团按钮。
- 订阅团长按钮。

**功能抽象：**

| 功能 | 说明 |
|---|---|
| 商品推荐流 | 按用户兴趣、关注团长、热度等展示商品 |
| 分类筛选 | 按商品类目筛选团购 |
| 频道切换 | 全部、视频、最新等内容维度切换 |
| 商品搜索 | 搜索商品名、团购名 |
| 商品卡片跳转 | 点击进入团购详情页 |
| 跟团快捷入口 | 从列表直接进入购买链路 |
| 订阅团长 | 关注某个团长后继续接收该团长商品 |
| 领券 | 进入平台或团长优惠券领取入口 |
| 购物车 | 查看已加入购物车的团购商品 |

### 5.2 团长主页 / 店铺主页

团长主页是买家看到的经营前台，页面名称和展示语言更偏“团长”；但从经营配置看，它背后承载的是店铺能力。新截图显示，即使用户还没有发布任何团购，也会存在一个默认主页，页面中包含编辑主页、订阅邀请、客服、位置、帮卖介绍、相册素材号等入口。

**页面元素：**

- 团长头像、昵称、身份标签。
- 成员数、关注数、跟团数等关系和交易背书。
- 金牌团长、冠军团长、回复超快、回头客多等信任标签。
- 编辑主页按钮。
- 客服按钮，通常指向当前团长 / 店铺客服。
- 订阅 / 订阅邀请按钮。
- 地址 / 位置展示，例如“添加位置，让更多的人购买”。
- 更多信息入口。
- 公众号关注提醒：关注后接收活动、订单、物流通知。
- 新人立减优惠券、红包入口。
- 会员权益、积分商城。
- 帮卖介绍入口，可引导填写帮卖说明。
- 相册素材号入口，可能用于沉淀商品素材、图文素材或内容号。
- 团员晒单。
- 商品列表。
- 排序 Tab：默认、上新、销量。
- 销量维度下可出现近 30 天 / 近 7 天筛选。
- 团购状态：在抢、已结束、近期发布。
- 空状态：暂时还没有发布的团购。

**功能抽象：**

| 功能 | 说明 |
|---|---|
| 团长信息展示 | 展示团长头像、名称、标签、成员数、关注数等 |
| 店铺前台承载 | 虽然前台称团长主页，但实际承载店铺主页、商品列表、会员、积分和分销说明 |
| 默认主页生成 | 用户即使未发布团购，也可拥有默认团长主页和空状态页面 |
| 编辑主页 | 修改团长主页资料、介绍、背景图、位置、展示信息 |
| 关注 / 订阅团长 | 用户订阅后接收团长开团通知 |
| 订阅邀请 | 团长生成订阅邀请卡片或链接，分享到微信，沉淀私域用户 |
| 客服咨询 | 优先跳转当前团长 / 店铺客服；官方客服作为平台兜底 |
| 新人优惠 | 新用户领取团长新人券 |
| 商品列表 | 展示当前店铺/团长发布的所有团购，未发布时展示空状态 |
| 排序筛选 | 默认、上新、销量排序，销量下可做时间范围筛选 |
| 团员晒单 | 展示用户评价、晒单、购买反馈 |
| 会员权益 | 团长店铺会员折扣或权益 |
| 积分商城 | 跟团获得积分，可兑换权益或商品 |
| 帮卖介绍 | 团长对帮卖规则、供货优势、分销说明进行介绍 |

### 5.8 会员卡页

新截图中出现“会员卡”页面，说明快团团存在围绕团长店铺的会员权益体系。该页面不是单纯的平台会员卡，而是用户与不同团长之间的私域会员关系集合。

**页面元素：**

- 会员卡列表。

**功能抽象：**

| 功能 | 说明 |
|---|---|
| 团长会员卡 | 每张会员卡归属于一个团长或店铺 |
| 会员等级 | 通过 V0、V1 等等级表达用户与团长的关系深度 |

**业务价值：**

- 用成长值和折扣权益提升复购。
- 让用户在多个团长店铺之间形成不同会员身份。
- 为团长提供用户分层运营基础。

**数据建模建议：**

会员卡应拆成“用户 - 团长”的关系数据，而不是单一用户资产：

```text
用户 ID + 团长 ID + 会员等级 + 成长值 + 折扣权益 + 最近购买时间 + 累计消费金额
```

## 6. 团长端页面与功能

快团团的团长端不是独立后台，而是穿插在普通用户路径中的经营能力集合。用户可以从个人中心、团长主页、一键开团、创建店铺等多个入口进入经营流程。

结合新增截图，团长端更适合拆成四个层级：

```text
身份激活：普通用户拥有开团和开店入口
    ↓
店铺配置：创建店铺、编辑主页、设置物流、开启帮卖分销
    ↓
团购发布：填写介绍、添加商品、配置团购设置、发布团购
    ↓
经营增长：邀请订阅、开团通知、微信分享、订单履约、会员复购、帮卖佣金
```

这个模型能解释截图中的混用现象：买家端强调“团长”，开店和后台配置强调“店铺”，发布流程强调“团购”。

团购设置出现在发布流程中，用于配置本次团购活动的交易规则。新增截图显示，设置项里不仅有物流方式和团购时间，还单独出现了“发货时间”，说明“团购开始/结束时间”和“承诺发货时间”是两个不同概念。

| 库存展示权限 | 是否向跟团用户展示商品库存 |
| 开团通知对象 | 向订阅用户发送开团通知 |

**功能价值：**

- 控制团购传播范围。
- 防止商品信息被随意复制。
- 保护私域团购内容。
- 根据团长经营策略设置公开或私密销售。

### 6.7 设置帮卖 / 帮卖分销

帮卖是快团团中非常重要的分销能力。结合新增“帮卖分销设置”截图，帮卖既可以作为团购级设置出现，也可以作为店铺级能力开启。店铺开启帮卖分销后，可以把店铺商品设置为可帮卖商品，并设置佣金，让更多团长帮忙销售；同时自己也可以帮卖他人的商品。

**截图中可见规则：**

| 设置项 | 截图表现 | 说明 |
|---|---|---|
| 帮卖分销开关 | 已开启 / 绿色开关 | 店铺级分销能力，可开启或关闭 |
| 可帮卖商品 | 可设置店铺商品为可帮卖商品 | 说明帮卖不只依附单次团购，也依赖店铺商品资产 |
| 佣金说明 | 交易服务费由供货团长支付 | 供货方承担平台服务费或交易服务费 |
| 佣金类型 | 固定佣金 | 设置商品售价和佣金比例后，帮卖团长按规则获得佣金 |
| 佣金发放方式 | 实时结款 | 帮卖团长每卖出 1 单，即可立即获得 1 笔帮卖佣金 |
| 设置佣金比例 | 编辑商品或批量管理时设置 | 支持单品设置，也可能支持批量设置多个商品佣金比例 |
| 帮卖他人商品 | 也可帮卖他人的商品 | 团长既可以作为供货方，也可以作为帮卖方 |

**帮卖方式：**

| 帮卖方式 | 说明 |
|---|---|
| 复制我的团进行帮卖 | 帮卖团长复制原团内容后售卖 |
| 分享我的团进行帮卖 | 帮卖团长分享原团链接售卖 |
| 店铺商品帮卖 | 店铺将商品设置为可帮卖，其他团长基于商品参与分销 |
| 帮卖他人商品 | 当前团长作为帮卖方，帮助其他供货团长卖货 |

**佣金类型：**

| 佣金类型 | 说明 |
|---|---|
| 固定佣金 | 供货团长设置商品售价和佣金比例，帮卖团长卖出后获得相应佣金 |
| 团长自由定价 | 供货团长设置供货价，帮卖团长自由设置售价，收益为售价与供货价差额 |

**佣金发放方式：**

| 发放方式 | 说明 |
|---|---|
| 实时结款 | 帮卖团长每卖出 1 单，立即获得 1 笔帮卖佣金 |
| 手动发放 | 供货团长手动发放佣金，或在约定时间后自动发放 |

**角色拆分：**

| 角色 | 说明 |
|---|---|
| 供货团长 | 提供商品、设置售价/供货价、佣金比例、交易服务费承担方，并负责供货或履约 |
| 帮卖团长 | 负责分享、复制或销售商品，成交后获得佣金或差价收益 |

**功能价值：**

- 扩大团购传播范围。
- 让其他团长参与销售。
- 形成轻量分销网络。
- 提升商品曝光和成交量。
- 把“店铺商品”变成可被分发的供货资产。

### 6.10 店铺设置与团购设置的区别

新增截图中同时出现“创建店铺”和“发布团购设置”，容易让人混淆。建议在产品模型中明确区分：

| 设置类型 | 归属 | 生命周期 | 示例 |
|---|---|---|---|
| 店铺设置 | 店铺 / 小店 | 长期存在，影响整个经营主体 | 店铺名称、店铺头像、店铺简介、店铺物流、帮卖分销 |
| 团购设置 | 单次团购活动 | 随每一次团购创建和结束 | 物流方式、发货时间、团购时间、开团通知推送、优惠设置、隐私设置 |
| 商品设置 | 商品 / SKU | 可复用，也可被单次团购引用 | 名称、描述、图片、规格、价格、库存、佣金比例 |
| 主页设置 | 团长前台展示 | 长期存在，影响信任和转化 | 背景图、位置、更多信息、客服、订阅邀请、帮卖介绍 |

一个典型例子：

```text
用户“某绅士”
  ↓
激活为团长
  ↓
拥有“某绅士的小店”
  ↓
店铺默认物流为快递，帮卖分销已开启
  ↓
发布“山东蒙阴白玉蜜桃”团购
  ↓
本次团购设置 7 天后结束、选择发货时间、推送全部订阅成员
  ↓
团购里售卖“一箱 4.5–5 斤白玉蜜桃”，价格 20 元，库存不限
```

**结论：**

> 店铺是长期经营资产，团购是短期销售活动。团长负责建立信任，店铺负责承载配置，团购负责完成一次交易，商品负责被售卖。

### 6.11 客服归属补充

结合客服入口出现位置，可进一步推测：

- 商品详情、团购详情、团长主页里的“客服”，更可能是当前团长 / 当前店铺客服。
- 个人中心里的“官方客服”，更可能是平台级客服。
- 买家在具体交易问题上优先找团长 / 店铺客服；在支付异常、投诉举报、平台规则问题上再找官方客服。

产品设计上可以把客服归属设计为：

```text
客服会话
├── 关联用户
├── 关联团长
├── 关联店铺
├── 可选关联团购
├── 可选关联订单
└── 可选升级为平台客服工单
```

### 17.1 开团前引导与角色分流

### 17.4 隐私权限的精确枚举

原文已覆盖“隐私设置、隐藏团购、复制权限、库存展示权限”，但截图中的隐私设置选项可以更精确枚举：

| 隐私/转发配置 | 说明 |
|---|---|
| 所有人均可转发 | 默认开放传播，适合公开团购和裂变传播 |
| 只有发布者可转发 | 限制其他用户转发，适合私域或限量团 |
| 隐藏团购且所有人均可转发 | 不在公开列表展示，但拿到链接/入口的人可继续传播 |
| 隐藏团购且只有发布者可转发 | 既隐藏团购，又限制传播权限，适合强私密团购 |

建议在“6.6 设置权限”中把“隐私设置”拆成两个维度：

```text
可见性：公开 / 隐藏
传播权：所有人可转发 / 仅发布者可转发
```

这样后续做权限模型时更清晰。

**修正后的产品抽象：**

```text
基础用户
  ├── 作为团员：浏览、订阅、购买、会员成长、复购
  └── 作为潜在团长：创建店铺、编辑主页、邀请订阅、发布团购、帮卖分销
```

这意味着快团团的真实增长逻辑不是单一的“团长卖货”，而是：

> 买家可以被自然转化为团长，团长也仍然保留买家身份。平台通过个人中心、一键开团、创建店铺、会员卡、订阅邀请，把用户的消费关系、经营关系和分销关系合并到同一个账号体系里。

**对系统设计的影响：**

| 设计项 | 影响 |
|---|---|
| 账号体系 | 用户表与团长表不能完全割裂，团长应是用户的扩展身份 |
| 权限系统 | 需要支持“未开店 / 已开店 / 已发团 / 帮卖中”等状态型权限 |
| 页面导航 | 个人中心必须同时承载买家资产和团长经营入口 |
| 数据模型 | 需要新增店铺、团长主页、订阅关系、会员关系、成长值等对象 |
| 增长策略 | 开店、订阅邀请、会员卡、帮卖分销共同驱动用户转团长和复购 |
| MVP 排期 | 创建店铺和默认团长主页应进入早期版本，而不是后期商家后台能力 |

## 19. 团长、店铺、团购关系的最终修订

基于最新补充截图，原先“用户即团长”的判断需要进一步精细化：

> 快团团确实弱化了买家和卖家的边界，但它并不是把用户、团长、店铺、团购完全做成同一个对象，而是在同一账号体系下逐层激活经营能力。

### 19.1 最终关系模型

```text
用户账号
  ├── 买家能力：浏览、订阅、购买、订单、售后、会员卡
  └── 经营能力
        ↓
      团长身份：前台信任主体
        ↓
      店铺 / 小店：长期经营容器
        ↓
      团购活动：一次销售活动
        ↓
      商品 / SKU：具体售卖对象
```

### 19.2 为什么会看起来混乱

快团团在不同场景使用了不同语言：

| 场景 | 产品说法 | 实际含义 |
|---|---|---|
| 买家浏览 | 团长主页、金牌团长、冠军团长、客服、订阅 | 强调人和信任关系 |
| 个人中心 | 店铺、团员、社群、会员卡 | 把普通用户引导成经营者 |
| 创建店铺 | 店铺名称、店铺头像、店铺简介、店铺物流、帮卖分销 | 配置长期经营容器 |
| 发布团购 | 介绍、商品、设置 | 创建一次销售活动 |
| 帮卖分销 | 店铺商品、供货团长、帮卖团长、佣金 | 把商品和团长网络连接起来 |

因此，调研文档中应避免简单写“团长就是店铺”或“用户就是团长”。更准确的表述是：

> 用户可以成为团长，团长拥有店铺，店铺发布团购，团购承载商品；买家感知的是团长信任，经营者配置的是店铺资产。

### 19.3 对产品设计的启发

如果要仿照快团团设计类似系统，建议从一开始就拆清楚对象边界：

| 系统对象 | 是否必须独立建表 / 独立模块 | 原因 |
|---|---|---|
| 用户 | 是 | 承载微信账号、买家行为和基础资料 |
| 团长身份 | 是 | 一个用户是否激活经营能力、团长标签和信任数据需要独立维护 |
| 店铺 | 是 | 店铺名称、头像、简介、物流、帮卖分销等是长期经营配置 |
| 团购活动 | 是 | 每次团购都有独立介绍、时间、发货、通知、优惠和权限 |
| 商品 | 是 | 商品可复用、可进入商品库、可被多个团购引用 |
| 团购商品关系 | 是 | 解决商品在不同团购中价格、库存、排序不同的问题 |
| 客服会话 | 建议独立 | 需要区分团长客服、店铺客服、官方客服和关联订单 |
| 订阅关系 | 是 | 开团通知和私域复购的核心数据 |
| 会员关系 | 是 | 会员卡是用户与团长/店铺之间的关系，不是单纯用户资产 |

### 19.4 更新后的完整性判断

更新后，当前调研已覆盖：

- 买家端浏览、下单、订单、消息、个人中心。
- 团长端开店、主页、订阅邀请、一键开团、发布团购。
- 店铺配置：名称、头像、简介、物流、帮卖分销。
- 团购发布三段式：介绍、商品、设置。
- 商品配置：名称、描述、图片、规格、价格、库存、更多设置。
- 团购设置：物流方式、发货时间、团购时间、开团通知推送、更多团购设置。
- 帮卖分销：供货团长、帮卖团长、固定佣金、实时结款、交易服务费承担。
