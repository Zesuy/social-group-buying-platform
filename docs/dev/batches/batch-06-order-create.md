# Batch 6：创建订单，地址快照、商品快照、订单明细

## 1. 目标

用户提交订单并生成订单主表、地址快照和商品快照。

## 2. 前置条件

Batch 5 已完成；买家可管理地址并完成订单预览。

## 3. 涉及接口

`POST /api/v1/orders`；`GET /api/v1/my/orders`；`GET /api/v1/my/orders/{orderId}`；`POST /api/v1/orders/{orderId}/cancel`。

## 4. 涉及数据表

`orders`、`order_items`、`addresses`、`group_buys`、`group_buy_items`、`products`。

## 5. 实现任务

创建订单时再次校验团购、商品、地址、库存；保存地址快照和商品快照；订单初始为 `orderStatus=pendingPay`、`payStatus=unpaid`；实现我的订单列表、详情和待支付取消。

## 6. 测试要求

MockMvc 覆盖创建订单、订单列表、详情、取消、越权访问、库存不足；Service 覆盖地址快照、商品快照、创建订单事务、取消状态校验。

## 7. 验收标准

创建订单不扣减库存；历史订单不受地址或商品后续修改影响。

## 8. 联调文档更新项

完成“创建订单”链路，补充订单详情页和取消订单的联调说明。

## 9. 禁止事项

不实现非本批接口；不实现模拟支付、真实支付、优惠抵扣、退款售后；创建订单阶段不扣减库存；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
