# Batch 1：模拟登录与当前用户

## 1. 目标

完成 MVP 登录态、当前用户识别和用户基础数据。

## 2. 前置条件

Batch 0 已完成；统一响应、错误码、认证基础、`users`、`leaders`、`stores` 表已可用。

## 3. 涉及接口

`POST /api/v1/auth/mock-login`；`GET /api/v1/me`。

## 4. 涉及数据表

`users`、`leaders`、`stores`。

## 5. 实现任务

实现模拟登录、用户查找或创建、accessToken 生成和解析、当前用户上下文；`GET /me` 返回 user、leader、store 摘要。

## 6. 测试要求

MockMvc 覆盖模拟登录、获取当前用户、未登录访问、无效 token；Service 覆盖用户创建 / 查询和当前身份组装。

## 7. 验收标准

前端可保存 `accessToken` 并通过 `GET /api/v1/me` 恢复登录态；未登录接口返回 `UNAUTHORIZED`。

## 8. 联调文档更新项

完成“模拟登录与获取当前用户”链路的请求样例、响应样例、错误码和测试数据准备方式。

## 9. 禁止事项

不实现非本批接口；不实现真实微信登录；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。

- 已更新联调文档的位置。

- 本批未解决问题，如无则写“无”。
