# Batch 9：订阅团长与会员卡展示

## 1. 目标

用户可订阅团长，并查看基础会员卡。

## 2. 前置条件

Batch 8 已完成；公共浏览接口已存在，会员关系可在模拟支付后创建或更新。

## 3. 涉及接口

`POST /api/v1/leaders/{leaderId}/subscription`；`DELETE /api/v1/leaders/{leaderId}/subscription`；`GET /api/v1/my/subscriptions`；`GET /api/v1/my/member-cards`。

## 4. 涉及数据表

`subscriptions`、`member_relations`、`leaders`、`stores`。

## 5. 实现任务

实现订阅、取消订阅、我的订阅列表；实现会员卡列表展示；明确重复订阅策略并保持接口一致；更新公共浏览接口中的 `viewer.subscribed`，使已登录用户返回真实订阅状态，未登录用户继续返回 `false`。

## 6. 测试要求

MockMvc 覆盖订阅成功、取消订阅、重复订阅、未登录、会员卡列表；Service 覆盖唯一关系和会员卡聚合；补充公共浏览接口已登录订阅状态测试。

## 7. 验收标准

团长主页和团购详情中的 `viewer.subscribed` 与订阅关系一致；支付成功后的会员关系可在会员卡页展示。

## 8. 联调文档更新项

完成“订阅团长”和“会员卡展示”链路，补充空状态和已有会员关系数据；回填“公共浏览团购”链路中的真实订阅状态说明。

## 9. 禁止事项

不实现非本批接口；不实现公众号推送、订阅邀请卡、订阅红包、会员等级升级规则、折扣计算、积分商城；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
