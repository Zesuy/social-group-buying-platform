# Batch 2：创建店铺与激活团长身份

## 1. 目标

用户创建店铺后自动生成团长身份，形成经营入口。

## 2. 前置条件

Batch 1 已完成；用户可模拟登录并通过 token 获取当前用户。

## 3. 涉及接口

`POST /api/v1/stores`；`GET /api/v1/my/store`；`PATCH /api/v1/my/store`。

## 4. 涉及数据表

`users`、`leaders`、`stores`。

## 5. 实现任务

实现一个用户最多一个团长身份、一个团长最多一个店铺；创建店铺时创建 leader 和 store；实现我的店铺查询和更新；校验 `/my/store` 均从 token 获取当前店铺。

## 6. 测试要求

MockMvc 覆盖创建、查询、更新、未登录、重复创建、参数错误；Service 覆盖唯一约束和重复创建异常。

## 7. 验收标准

创建成功后 `GET /api/v1/me` 中 `hasLeader=true`，并返回 `leaderId`、`storeId`；重复创建返回 `STORE_ALREADY_EXISTS`。

## 8. 联调文档更新项

完成“创建店铺”链路，补充创建成功后跳转团长主页或发布入口的判断方式。

## 9. 禁止事项

不实现非本批接口；不实现平台后台或店铺认证；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
