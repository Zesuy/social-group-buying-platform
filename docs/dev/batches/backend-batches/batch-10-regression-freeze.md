# Batch 10：全链路回归、接口文档冻结、联调准备

## 1. 目标

冻结 MVP 接口口径，完成端到端业务回归，为前端集中联调做准备。

## 2. 前置条件

Batch 0 到 Batch 9 均已完成；所有 MVP 接口和联调链路已有初始说明。

## 3. 涉及接口

MVP 全部接口。

## 4. 涉及数据表

MVP 全部表：`users`、`leaders`、`stores`、`products`、`group_buys`、`group_buy_items`、`addresses`、`orders`、`order_items`、`shipments`、`subscriptions`、`member_relations`。

## 5. 实现任务

梳理接口与 `docs/API设计.md` 是否一致；补齐缺失错误码；整理测试数据脚本或说明；执行从登录、开店、发团、浏览、下单、支付、发货、会员卡的全链路回归。

## 6. 测试要求

全量单元测试和 MockMvc 测试通过；补充核心链路集成测试；覆盖权限、状态、库存、快照、重复支付、重复发货。

## 7. 验收标准

MVP 主链路可在空库初始化后跑通；接口文档和联调文档无已知差异；无已知问题被留到下一批。

## 8. 联调文档更新项

冻结 `docs/前后端联调文档.md` 的 MVP 链路说明，标记可联调接口、测试账号和测试数据准备步骤。

## 9. 禁止事项

不实现非 MVP 新功能；不新增真实微信支付、优惠券、售后退款、购物车、帮卖分销、积分商城、公众号推送、平台后台；不跳过测试；不把已知问题留到后续阶段；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
