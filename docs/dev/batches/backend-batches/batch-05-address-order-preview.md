# Batch 5：地址管理与订单预览

## 1. 目标

用户可维护地址，并在下单确认页完成订单预览。

## 2. 前置条件

Batch 4 已完成；买家可登录，系统中存在可购买团购。

## 3. 涉及接口

`GET /api/v1/my/addresses`；`POST /api/v1/my/addresses`；`PATCH /api/v1/my/addresses/{addressId}`；`DELETE /api/v1/my/addresses/{addressId}`；`POST /api/v1/orders/preview`。

## 4. 涉及数据表

`addresses`、`group_buys`、`group_buy_items`、`products`。

## 5. 实现任务

实现地址增删改查、默认地址；订单预览校验团购可购买、商品属于团购、地址属于当前用户、库存充足；预览只计算金额，不创建订单、不扣库存。

## 6. 测试要求

MockMvc 覆盖地址 CRUD、地址越权、预览成功、团购结束、商品不属于团购、地址不属于用户、库存不足；Service 覆盖金额计算。

## 7. 验收标准

下单确认页可拿到地址、商品明细、`totalAmount`、`discountAmount=0`、`payAmount`。

## 8. 联调文档更新项

完成“地址管理与订单预览”链路，补充地址准备和库存不足处理。

## 9. 禁止事项

不实现非本批接口；不创建订单、不扣库存、不占库存；不实现地址智能解析或地址风控；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
