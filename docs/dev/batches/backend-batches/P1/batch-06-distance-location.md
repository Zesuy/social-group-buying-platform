# P1 Batch 6：店铺定位和距离展示

## 1. 目标

为店铺增加 GPS 经纬度存储能力，并在公共浏览接口中支持基于用户当前位置计算和展示距离。前端可通过浏览器定位获取用户经纬度并传给后端；后端按统一算法返回距离字段，必要时支持按距离筛选或排序。

本批只做轻量位置能力，不把距离作为配送承诺或下单限制。

## 2. 前置条件

P1 Batch 1 到 Batch 5 均已完成；店铺、商品、团购、订单、售后和 P1 回归链路稳定。公共团购列表、团购详情、店铺信息和团长店铺管理接口已具备兼容扩展字段的基础。

## 3. 涉及接口

增强 `POST /api/v1/stores`；增强 `GET /api/v1/my/store`；增强 `PATCH /api/v1/my/store` 或既有店铺更新接口；增强 `GET /api/v1/group-buys`；增强 `GET /api/v1/group-buys/{groupBuyId}`；如已有店铺公开详情接口则增强对应 `GET /api/v1/stores/{storeId}`。

公共浏览接口允许可选查询参数：

- `latitude`：用户当前位置纬度，范围 `[-90, 90]`。
- `longitude`：用户当前位置经度，范围 `[-180, 180]`。
- `maxDistanceMeters`：可选，按距离筛选，必须为正整数。
- `sort=distance`：可选，按距离由近到远排序；仅在同时传入合法 `latitude` 和 `longitude` 时生效。

响应中为店铺或团购所属店铺增加可选字段：

- `storeLatitude` / `storeLongitude`：仅在接口原本需要返回店铺定位信息时返回；不应在不需要定位的私有场景过度暴露。
- `distanceMeters`：用户位置与店铺位置的直线距离，单位米；用户未授权定位、未传用户坐标、店铺无坐标或坐标非法时返回 `null`。
- `distanceText`：展示文案，如 `800m`、`1.2km`；也可由前端根据 `distanceMeters` 统一格式化，但本批必须在 API 文档中固定口径。

## 4. 涉及数据表

增强 `stores` 表，新增可空字段：

- `latitude decimal(10,7)`：店铺纬度。
- `longitude decimal(10,7)`：店铺经度。

如果当前店铺表已有地址字段，不改变既有地址语义；经纬度只表示团长维护的店铺或提货点位置。历史店铺数据允许经纬度为空。

## 5. 实现任务

店铺创建和更新时允许团长填写或修改经纬度，并校验数值范围；店铺详情和团长自己的店铺信息能返回已保存的经纬度；公共团购列表和详情在收到用户经纬度后，使用 Haversine 公式计算用户与店铺的直线距离；公共列表支持可选的距离筛选和距离排序，未传用户坐标时不启用距离筛选或排序；店铺无坐标的团购在普通列表中仍可展示，但在 `maxDistanceMeters` 筛选下应被排除。

距离计算只用于展示和发现效率，不影响订单预览、下单、支付、发货、售后、优惠券和会员成长。定位失败、用户拒绝授权或浏览器不支持定位时，前端不应阻断浏览和下单，后端接口也必须兼容不传坐标的请求。

## 6. 测试要求

MockMvc 覆盖店铺创建带经纬度、店铺更新经纬度、经纬度越界校验、公共团购列表不传坐标、公共团购列表传坐标返回距离、按距离筛选、按距离排序、店铺无坐标时距离为 `null`、未登录公共浏览、非团长修改店铺拒绝、跨店铺修改拒绝。

Service 覆盖 Haversine 距离计算、米 / 千米展示口径、非法坐标处理、距离筛选边界值、同距离排序稳定性、历史店铺无坐标兼容。

如距离排序在数据库层实现，必须补充 Mapper 或 Repository 测试，覆盖分页、距离字段和普通排序共存。

## 7. 验收标准

店铺可以保存和更新合法经纬度；公共团购列表和详情在用户坐标合法时返回稳定的距离数据；定位参数缺失或非法不会影响原有公共浏览；按距离筛选和排序只在坐标合法时生效；已有 MVP 和 P1 交易链路不因定位字段产生兼容性回归。

## 8. 联调文档更新项

新增“前端定位与距离展示”链路；更新店铺创建 / 更新、公共团购列表、团购详情的请求样例、响应样例、定位失败降级方式、常见错误码和测试数据准备方式。明确前端通过浏览器 Geolocation API 获取用户坐标，坐标只作为本次请求参数传递，不要求后端存储用户实时位置。

## 9. 禁止事项

不接地图 SDK、不做逆地理编码、不做行政区解析、不实现配送范围判定、不因距离远近限制下单、不存储用户实时 GPS 轨迹、不实现附近门店推荐算法、不改变订单、支付、发货、售后、优惠券和会员成长金额口径。
