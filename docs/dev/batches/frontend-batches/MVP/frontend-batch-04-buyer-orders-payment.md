# Frontend Batch 04：买家订单列表、订单详情、模拟支付与确认收货

## 1. 目标

完成买家订单生命周期页面：查看订单、模拟支付、取消订单、确认收货。

## 2. 前置条件

Frontend Batch 03 已完成；买家能够创建订单并跳转订单详情；后端订单查询和状态流转接口可用。

## 3. 涉及接口

- `GET /api/v1/my/orders?status=xxx&page=1&pageSize=20`
- `GET /api/v1/my/orders/{orderId}`
- `POST /api/v1/orders/{orderId}/simulate-pay`
- `POST /api/v1/orders/{orderId}/cancel`
- `POST /api/v1/orders/{orderId}/complete`

## 4. 页面范围

- `/orders`
- `/orders/:id`

## 5. 实现任务

实现订单列表页，包含搜索占位、状态 Tab、分页加载、订单卡片、空状态和错误重试；订单卡片根据状态展示“去支付”“取消订单”“确认收货”等操作；实现订单详情页，展示订单状态步骤条、地址快照、商品快照、金额明细、订单号、支付时间、发货时间、完成时间；实现模拟支付、取消订单、确认收货操作，均使用确认弹窗、loading、disabled 和错误提示；支付成功后刷新详情并支持返回列表查看状态变化。

## 6. 测试要求

组件测试覆盖订单卡片不同状态按钮、订单状态步骤条、金额明细；单元测试覆盖订单状态文案映射和按钮可见性；E2E 覆盖创建订单后模拟支付、取消待支付订单、已发货订单确认收货；异常路径覆盖重复支付、不可取消、不可确认收货的错误提示。

## 7. 验收标准

买家可以查看自己的订单列表和详情；待支付订单可以模拟支付或取消；已发货订单可以确认收货；订单状态、支付状态和操作按钮一致；后端返回的重复操作和状态错误能被清晰提示。

## 8. 联调文档更新项

更新 `docs/前后端联调文档.md` 中买家订单、模拟支付、取消订单、确认收货链路，补充页面入口、状态 Tab、操作按钮、成功判定和常见错误码。

## 9. 禁止事项

不接真实微信支付；不实现售后申请、审核或退款；不做订单搜索真实接口；不做猜你喜欢真实推荐；不在前端伪造支付成功状态。
