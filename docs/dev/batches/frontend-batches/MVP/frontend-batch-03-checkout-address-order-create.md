# Frontend Batch 03：地址管理、下单预览与创建订单

## 1. 目标

完成买家从团购详情进入下单确认、管理收货地址、预览金额并创建订单的链路。

## 2. 前置条件

Frontend Batch 02 已完成；团购详情页已有购买入口、数量选择和登录拦截；后端地址、订单预览和创建订单接口可用。

## 3. 涉及接口

- `GET /api/v1/my/addresses`
- `POST /api/v1/my/addresses`
- `PATCH /api/v1/my/addresses/{addressId}`
- `DELETE /api/v1/my/addresses/{addressId}`
- `POST /api/v1/orders/preview`
- `POST /api/v1/orders`

## 4. 页面范围

- `/checkout`
- `/addresses`
- `/addresses/new`
- `/addresses/:id/edit`

## 5. 实现任务

实现 `checkoutStore` 保存当前下单上下文，注意它不是购物车；实现地址列表、新增地址、编辑地址、删除确认、默认地址展示和从下单页选择地址后返回；实现 `AddressCard` 和地址表单校验；实现下单确认页，展示地址卡片、商品明细、数量加减、备注、协议勾选、金额区和底部固定操作栏；进入下单页后先调用订单预览，不创建订单；切换地址或数量后重新预览；提交订单按钮必须 loading、disabled、防重复点击；创建成功后跳转订单详情页。

## 6. 测试要求

单元测试覆盖下单金额展示、地址表单校验、协议勾选校验；组件测试覆盖地址卡片默认状态、无地址空态、下单确认金额区和提交按钮防重复状态；E2E 覆盖登录后新增地址、从团购详情进入下单页、预览订单、创建订单成功跳转。

## 7. 验收标准

买家可以完整创建地址并完成订单创建；无地址时有明确引导；预览和创建订单请求体符合冻结 API；库存不足、地址不属于当前用户、团购不可购买等错误能正常提示；重复点击不会产生多次创建请求。

## 8. 联调文档更新项

更新 `docs/前后端联调文档.md` 中地址管理、下单预览、创建订单链路，补充前端入口、从详情页带入的参数、请求样例、成功跳转和常见错误码。

## 9. 禁止事项

不实现购物车；不接真实微信支付；不做优惠抵扣；不把提交订单等同于支付成功；不允许前端使用浮点金额；不绕过后端预览结果自行计算应付金额作为最终依据。
