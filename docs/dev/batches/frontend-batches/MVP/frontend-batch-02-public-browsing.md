# Frontend Batch 02：公共浏览、团购详情、团长主页与订阅操作

## 1. 目标

完成未登录可浏览的公开内容链路：首页团购信息流、团购详情、团长主页，并接入登录后订阅 / 取消订阅能力。

## 2. 前置条件

Frontend Batch 01 已完成；登录拦截、API client、错误提示和基础卡片组件可复用。

## 3. 涉及接口

- `GET /api/v1/group-buys?page=1&pageSize=20`
- `GET /api/v1/group-buys/{groupBuyId}`
- `GET /api/v1/leaders/{leaderId}/homepage?page=1&pageSize=20`
- `POST /api/v1/leaders/{leaderId}/subscription`
- `DELETE /api/v1/leaders/{leaderId}/subscription`

## 4. 页面范围

- `/`
- `/group-buys/:id`
- `/leaders/:id`

## 5. 实现任务

实现首页顶部品牌区、提醒横幅、频道 Tab、分类横滑和团购卡片流；封装 `GroupBuyFeedCard`、`PriceText`、`ImageWithFallback`、`LeaderTrustBlock`、`StoreSummaryCard`、`ChannelTabs`、`CategoryChips`；首页支持下拉刷新、触底加载、空状态和错误重试；实现团购详情页，展示封面、团长 / 店铺信任区、标题、介绍、价格、配送方式、发货时间、结束时间、商品列表、数量选择和底部固定购买栏；实现团长主页，展示团长信息、店铺摘要、信任信息、订阅按钮和团购列表；订阅 / 取消订阅统一走登录拦截；分享、收藏、客服、积分商城等仅做占位或 toast，不接真实能力。

## 6. 测试要求

组件测试覆盖团购卡片、价格展示、空状态、详情页库存不足和已结束禁用状态；单元测试覆盖分页 composable；登录拦截测试覆盖未登录点击订阅和购买入口；E2E 覆盖未登录浏览首页、进入团购详情、进入团长主页。

## 7. 验收标准

未登录用户可以浏览首页、团购详情和团长主页；页面形态接近小程序信息流，突出商品图、价格、团长和店铺信息；订阅操作未登录时进入登录流程，登录后能完成订阅或取消订阅；非 MVP 入口不触发真实后端调用。

## 8. 联调文档更新项

更新 `docs/前后端联调文档.md` 中公共浏览、团长主页、订阅 / 取消订阅相关链路，补充前端页面路径、UI 入口、分页加载方式、成功判定和错误提示。

## 9. 禁止事项

不做真实搜索；不做真实分类筛选；不做购物车；不做优惠券 / 红包；不做真实收藏；不生成分享海报；不新增后端未定义字段作为必需依赖。
