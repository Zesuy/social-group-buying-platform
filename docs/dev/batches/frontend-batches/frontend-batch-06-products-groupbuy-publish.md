# Frontend Batch 06：轻量商品管理与三段式发布团购

## 1. 目标

完成团长发布普通团购前所需的轻量商品管理，并实现“介绍 / 商品 / 设置”三段式发布团购链路。

## 2. 前置条件

Frontend Batch 05 已完成；用户可以创建店铺并具备团长身份；团长路由守卫可用；后端商品和团购发布接口可用。

## 3. 涉及接口

- `GET /api/v1/my/store/products`
- `POST /api/v1/my/store/products`
- `GET /api/v1/my/store/products/{productId}`
- `PATCH /api/v1/my/store/products/{productId}`
- `DELETE /api/v1/my/store/products/{productId}`
- `POST /api/v1/my/store/group-buys`

## 4. 页面范围

- `/leader/products`
- `/leader/products/new`
- `/leader/products/:id/edit`
- `/leader/group-buys/new`

## 5. 实现任务

实现轻量商品列表、新建商品、编辑商品和删除确认，商品管理只服务发布团购复用，不做完整商品库；封装 `ProductForm`，字段包括商品名称、描述、封面图、基础价、库存和状态；实现发布团购三段式页面，包含“介绍”“商品”“设置”三个 Tab；介绍 Tab 包含团购标题、介绍、封面图 URL；商品 Tab 支持选择已有商品或内联创建商品，维护展示名、团购价、团购库存、排序；设置 Tab 包含配送方式、发货时间、开始时间、结束时间和协议勾选；发布按钮统一校验必填项、金额整数分、库存非负、时间顺序和协议；发布成功进入我的团购详情或团长主页。

## 6. 测试要求

单元测试覆盖商品表单校验、金额元 / 分转换、发布团购请求组装、时间顺序校验；组件测试覆盖商品列表空态、删除确认、三段式发布表单校验；E2E 覆盖团长新建商品、发布普通团购、从商品 Tab 选择已有商品发布团购。

## 7. 验收标准

团长可以维护轻量商品；发布团购页面形态为三段式；至少一个团购商品才允许发布；`groupStock` 不允许小于 0，不使用 `-1` 表示不限库存；发布请求只创建普通团购并符合冻结 API。

## 8. 联调文档更新项

更新 `docs/前后端联调文档.md` 中商品管理和发布普通团购链路，补充页面入口、三段式表单、请求样例、成功跳转、校验规则和常见错误码。

## 9. 禁止事项

不做完整商品库；不做分类管理；不做批量导入；不做素材库；不实现预售 / 卡券 / 报名发布；不做优惠设置、帮卖设置、隐私权限细分、复制团购、保存并预览或开团通知推送。
