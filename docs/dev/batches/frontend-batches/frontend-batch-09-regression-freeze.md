# Frontend Batch 09：全链路回归、E2E、构建检查与前端冻结

## 1. 目标

完成前端 MVP 全链路回归、E2E 覆盖、构建检查和联调文档收口，冻结可交付的前端 MVP。

## 2. 前置条件

Frontend Batch 00 到 Frontend Batch 08 均已完成；后端 Batch 10 已冻结；所有 MVP 页面和链路已有初始联调说明。

## 3. 涉及接口

MVP 前端已接入的全部 `/api/v1/**` 接口，包括认证、公共浏览、店铺、商品、团购、地址、订单、团长订单、订阅和会员卡。

## 4. 页面范围

前端 MVP 全部页面：

- 5 个底部 Tab：首页、订单、一键开团、消息、个人中心
- 公共浏览：团购详情、团长主页
- 买家链路：登录、地址、下单、订单列表、订单详情
- 团长链路：创建店铺、我的店铺、商品管理、发布团购、团购管理、团长订单、发货
- 私域关系：我的订阅、会员卡

## 5. 实现任务

按 `docs/dev/frontend-mvp-plan-revised.md` 的页面验收清单逐项检查；补齐缺失 loading、disabled、空态、错误态、安全区和移动端适配；统一检查金额、状态、时间、错误码和接口字段是否与冻结文档一致；清理临时调试代码和未使用依赖；补齐关键 E2E 场景；检查非 MVP 入口无真实调用；更新 `docs/前后端联调文档.md` 中所有前端入口、调用顺序、成功判定、测试数据准备方式和已知问题。

## 6. 测试要求

至少运行：

```bash
cd frontend
npm run typecheck
npm run lint
npm run test:unit
npm run test:e2e
npm run build
```

E2E 至少覆盖未登录浏览首页和团购详情、登录拦截后恢复购买操作、买家创建地址并下单、模拟支付、团长创建店铺并发布普通团购、团长发货、买家确认收货、订阅 / 取消订阅、会员卡展示、消息页空态和非 MVP 入口置灰。

## 7. 验收标准

前端 MVP 在空库测试数据准备后可跑通完整闭环；所有测试和构建通过；联调文档与实际页面入口和接口调用一致；无已知阻断问题留到后续阶段；非 MVP 能力没有被实现或误接入。

## 8. 联调文档更新项

冻结 `docs/前后端联调文档.md` 的前端联调说明，标记可联调页面、前置条件、调用顺序、请求样例、响应样例、成功判定、常见错误码和测试数据准备步骤。

## 9. 禁止事项

不新增 MVP 外功能；不临时修改后端冻结 API；不跳过失败测试；不把已知问题留到下一阶段；不为了通过 E2E 而绕过真实接口和状态流转；不隐藏非 MVP 入口的真实调用风险。

## 10. 本批完成后必须输出的结果

- 已实现页面 / 能力清单。
- 已新增 / 修改文件清单。
- 已新增 / 修改测试清单。
- 已更新联调文档的位置。
- 测试运行命令和结果。
- 本批未解决问题，如无则写“无”。
