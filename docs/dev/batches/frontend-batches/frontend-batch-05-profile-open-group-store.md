# Frontend Batch 05：个人中心、一键开团、创建店铺与店铺资料

## 1. 目标

完成普通用户成为团长的前端链路：个人中心识别身份、一键开团引导、创建店铺、维护自己的店铺资料。

## 2. 前置条件

Frontend Batch 04 已完成；登录态、当前用户、底部 Tab 和买家核心链路稳定；后端店铺创建和我的店铺接口可用。

## 3. 涉及接口

- `GET /api/v1/me`
- `POST /api/v1/stores`
- `GET /api/v1/my/store`
- `PATCH /api/v1/my/store`

## 4. 页面范围

- `/profile`
- `/open-group`
- `/store/create`
- `/leader/store`

## 5. 实现任务

实现个人中心，展示用户头像、昵称、身份标签、我的订单、会员卡、地址管理、退出登录，以及团长经营入口；未开店展示“创建店铺 / 成为团长”，已开店展示“我的店铺”“发布团购”“我的团购”“团长订单”；实现一键开团页，展示提醒横幅、开团类型卡片，普通团购可点击，预售 / 卡券 / 报名置灰并提示后续开放；点击普通团购时，未登录进入登录拦截，已登录未开店进入创建店铺，已开店弹出动作面板并进入发布团购页占位；实现创建店铺页和我的店铺资料页，支持店铺名称、logo URL、简介、默认物流方式；创建或保存成功后刷新 `GET /api/v1/me`。

## 6. 测试要求

组件测试覆盖个人中心未登录、买家、团长三种状态；组件测试覆盖一键开团类型卡片可用 / 置灰状态；单元测试覆盖根据 `me` 判断跳转目标的逻辑；E2E 覆盖未登录点击普通团购进入登录、已登录未开店进入创建店铺、创建店铺后出现团长经营入口。

## 7. 验收标准

用户可以从个人中心或一键开团进入创建店铺流程；创建成功后当前用户身份刷新为团长；已开店用户可以查看和编辑自己的店铺资料；非 MVP 开团类型始终不可进入真实流程。

## 8. 联调文档更新项

更新 `docs/前后端联调文档.md` 中个人中心、一键开团、创建店铺、我的店铺资料链路，补充页面入口、身份判断、调用顺序、成功判定和 `STORE_ALREADY_EXISTS` 等错误码展示。

## 9. 禁止事项

不实现预售团购、卡券团购、报名团购；不做真实图片上传；不做店铺认证；不做帮卖分销设置；不使用 `storeId` 直接拼接操作他人店铺资源，店铺管理统一走 `/my/store/**`。
