# Batch 4：公共浏览，首页团购列表、团购详情、团长主页

## 1. 目标

未登录用户可浏览首页团购、团购详情和团长主页。

## 2. 前置条件

Batch 3C 已完成；系统中存在已发布普通团购。

## 3. 涉及接口

`GET /api/v1/group-buys`；`GET /api/v1/group-buys/{groupBuyId}`；`GET /api/v1/leaders/{leaderId}/homepage`。

## 4. 涉及数据表

`leaders`、`stores`、`group_buys`、`group_buy_items`、`products`、`subscriptions`。

## 5. 实现任务

实现公开团购分页列表、详情聚合、团长主页聚合；如果订阅功能尚未实现，`viewer.subscribed` 暂时统一返回 `false`；Batch 9 完成订阅功能后，再更新公共浏览接口，使已登录用户返回真实订阅状态。

## 6. 测试要求

MockMvc 覆盖未登录浏览、已登录浏览、分页、下架或不存在资源；Service 覆盖只返回公开已发布团购；当前阶段覆盖 `viewer.subscribed=false` 的临时口径。

## 7. 验收标准

首页和详情页无需登录可用；响应包含价格、销量、团长、店铺和商品信息；订阅功能完成前不阻塞公共浏览联调。

## 8. 联调文档更新项

完成“公共浏览团购”链路，补充首页、详情页、团长主页调用顺序；注明 Batch 9 前 `viewer.subscribed=false`，Batch 9 后再改为真实订阅状态。

## 9. 禁止事项

不实现非本批接口；不提前实现订阅功能；不实现搜索、复杂分类、分享海报、消息推送；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
