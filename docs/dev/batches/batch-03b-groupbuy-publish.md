# Batch 3B：发布普通团购

## 1. 目标

团长可发布普通团购，支持复用已有商品或在发布时内联创建商品。

## 2. 前置条件

Batch 3A 已完成；团长可创建或选择自己店铺下的商品。

## 3. 涉及接口

`POST /api/v1/my/store/group-buys`。

## 4. 涉及数据表

`stores`、`products`、`group_buys`、`group_buy_items`。

## 5. 实现任务

实现普通团购创建并直接发布；MVP 仅支持 `groupType=normal`；一个团购至少包含一个团购商品；支持 item 传入 `productId` 复用当前店铺商品，或传入 `product` 内联创建商品；同一事务创建 `group_buys`、必要的 `products` 和 `group_buy_items`；校验商品归属、`groupStock >= 0`、团购价为整数分、时间字段合法。

## 6. 测试要求

MockMvc 覆盖发布成功、复用商品、内联商品、未登录、非团长、跨店铺商品拒绝、无商品、库存为负、参数错误；Service 覆盖发布事务、内联商品事务回滚、商品归属和团购商品库存校验。

## 7. 验收标准

团长可发布 `groupType=normal` 且 `status=published` 的团购；发布失败不能留下半成品团购、商品或团购商品关系；`groupStock` 不允许为负数或 `-1`。

## 8. 联调文档更新项

完成“发布普通团购”链路，补充内联创建商品和复用已有商品两种请求样例，以及失败回滚的联调验证点。

## 9. 禁止事项

不实现非本批接口；不实现预售团购、卡券团购、报名团购、隐藏团购、帮卖设置；不扩大 MVP 范围；不跳过测试；不把已知问题留到下一批；不私自修改 API 字段、状态枚举、错误码、金额口径；需要修改上游文档时，必须明确说明原因。
