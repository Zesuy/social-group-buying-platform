# 前端 AI 工作流

> 定位：本文约束 AI 执行前端页面、组件、交互和视觉改动时的工作方式。
>
> 目标：先对齐需求和方案，再写代码，减少页面返工和“看着能看但业务不对”的问题。

## 1. 适用范围

以下任务必须先走本文工作流：

- 新增页面；
- 重做页面；
- 调整页面主结构；
- 新增或重构核心组件；
- 修改首页、团购详情、团长主页、发布团购、订单等关键链路；
- 涉及 `DESIGN.md` 或 `docs/前端产品与页面设计准则.md` 的 UI 调整。

以下任务可以直接实现，但仍需遵守 `DESIGN.md`：

- 修复明确的样式 bug；
- 修改文案错别字；
- 调整单个已明确的间距、颜色、状态；
- 修复测试、类型或 lint 问题。

## 2. 工作流总览

前端页面级任务分三步：

```text
需求澄清 -> 前端技术方案 -> 实现与验收
```

在需求和方案未确认前，不写页面代码。

如果任务来自已经写清楚的历史 batch 文件，AI 仍需先在执行说明中完成需求复述和方案拆解；若发现边界不清、接口不明或与当前代码 / API 契约冲突，必须先核对实际实现并说明问题。

## 3. 第一步：需求澄清

需求澄清必须覆盖：

| 项目 | 要求 |
|---|---|
| 页面目标 | 这个页面解决什么问题 |
| 使用角色 | 买家、团长、未登录用户、已登录用户分别能做什么 |
| 页面入口 | 从首页、详情、个人中心、订单、分享链接等哪里进入 |
| 当前范围 | 本次做哪些能力，不做哪些能力 |
| 业务对象 | 团购、团长、店铺、商品、订单、订阅、会员等如何出现 |
| 数据字段 | 页面需要展示和提交哪些字段 |
| 接口依赖 | 使用哪些 API，请求时机是什么 |
| 权限边界 | 未登录、非团长、跨店铺、非资源所有者如何处理 |
| 状态场景 | loading、empty、error、disabled、success 如何处理 |
| 待确认问题 | 列出不能从文档或代码中确定的问题 |

需求澄清必须优先检查：

- 是否符合 `docs/功能需求定义.md`；
- 是否符合 `docs/API设计.md`；
- 是否符合 `docs/页面与交互文档.md`；
- 是否没有越过用户明确指定的页面、链路或功能范围；
- 是否没有强化未实现能力。

## 4. 第二步：前端技术方案

需求明确后，输出前端技术方案。

技术方案必须包含：

1. 页面职责和页面结构；
2. 页面区块顺序和首屏重点；
3. 组件拆分方案；
4. API 设计、字段映射和请求时机；
5. 数据流和状态流转；
6. loading、empty、error、disabled、success；
7. 权限和登录拦截；
8. 文件目录和改动清单；
9. 实现顺序；
10. 验收方式和测试命令。

方案必须说明：

- 哪些逻辑留在页面；
- 哪些抽成组件；
- 哪些抽成 composable / store / util；
- 哪些未实现能力隐藏或灰态；
- 如何遵守 `DESIGN.md` 的视觉系统。

## 5. 第三步：实现与验收

实现顺序建议：

```text
页面骨架
-> 数据类型和 API 调用
-> 状态和权限
-> 交互动作
-> 视觉细节
-> 单元 / E2E / 构建验证
-> 联调文档更新
```

不得为了视觉完整绕过真实接口、登录拦截、状态流转或当前范围边界。

完成后必须输出：

- 已实现页面 / 组件；
- 涉及接口；
- 新增 / 修改文件；
- 状态处理；
- 与 `DESIGN.md` 的符合点；
- 测试命令和结果；
- 未解决问题，如无写“无”。

## 6. 页面级 Prompt 模板

后续需要 AI 设计或实现页面时，优先使用以下模板：

```text
根据以下需求，先完成需求澄清和前端技术设计方案，不要写任何代码。

第一步：需求澄清
请列出页面目标、入口、角色、字段、接口、权限、状态、当前范围边界和待确认问题。

第二步：技术方案
在需求澄清后，输出：
1. 页面职责和页面结构
2. 区块顺序和首屏重点
3. 组件拆分方案
4. API 字段、请求时机和数据流
5. loading、empty、error、disabled、success
6. 权限和登录拦截
7. 文件目录和实现顺序
8. 验收清单和测试命令

必须遵守：
- AGENTS.md
- DESIGN.md
- docs/前端产品与页面设计准则.md
- docs/页面与交互文档.md
- 当前任务相关的历史 batch 文件（如需要）

需求：
[在此粘贴 PRD、batch 或口头需求]
```

## 7. 何时按页面推进

页面级工作流适合用于：

- 后端接口基本完成后的前端页面接入；
- 前端页面校准；
- 关键页面重做；
- 富功能阶段的单页增强。

在早期骨架阶段，可以按链路 batch 推进；但进入页面体验和视觉质量阶段后，应优先按页面或关键链路推进。

推荐前端校准顺序：

1. 首页；
2. 团购详情页；
3. 发布团购页；
4. 团长主页；
5. 个人中心；
6. 订单列表和订单详情；
7. 地址、会员卡、订阅等辅助页面。
