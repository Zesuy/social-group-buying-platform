# 私域团购核心页面审查与优化规划

> 本文记录一次围绕首页、团购详情、团长主页、登录承接和开团入口的页面专项审查。
> 目标是校准页面表达与 `docs/前端产品与页面设计准则.md`、`DESIGN.md` 的一致性，并形成后续前端落地计划。

## 1. 页面规划结论

本项目不是传统低价商城，也不是平台货架电商。首页和核心浏览链路应表达的是：

```text
团长/小商家组织一次团购活动
-> 用户基于团长信任、活动场景和履约承诺产生购买动机
-> 用户跟团下单
-> 团长集中履约
-> 用户订阅、复购
```

因此本轮优化的目标不是让页面更像促销商城，而是让页面更早、更清楚地呈现：

- 谁在组织这次团购；
- 这次团购卖什么；
- 为什么现在买；
- 什么时候结束或履约；
- 已有多少人参与；
- 如何跟团；
- 如何订阅团长并形成复购关系。

## 2. 页面级洞察

### 2.1 首页

当前首页方向基本正确，已经有平台身份、私域团购语境、团长/店铺、团购卡片、订阅和履约标签。

主要问题是信息优先级执行偏了：

- 首屏大面积用于平台理念、统计、三步流程、搜索、分类和定位提示；
- 第一张团购卡只露出部分内容，用户需要滚动后才能完整看到正在进行的团购；
- “一键开团 / 我要开团 / 底部开团 Tab”连续出现，经营入口对普通买家主线形成干扰；
- 团购卡片里的“团长履约 / 集中收单 / 社群分享”偏泛化，不足以形成具体购买动机；
- 第三张团购封面退化为文字色块，占位感会削弱生鲜团购的真实可信度。

首页应从“大 hero 讲平台”调整为“轻平台语境 + 立刻看到活动卡”。首屏必须完整露出至少一张团购活动卡。

### 2.2 团购详情页

团购详情页内容质量明显强于首页，已经包含：

- 团长 / 店铺信任区；
- 活动状态、标题、介绍；
- 热销商品；
- “这次团为什么开”；
- 商品列表；
- 履约说明；
- 底部购买栏。

主要问题是：

- 从首页滚动位置进入详情时，详情页可能继承上一页滚动位置，用户会错过顶部信任信息；
- 首屏信息量较大，热销商品、活动介绍和锚点区之间的主次关系可以更清楚；
- 详情页优势信息没有充分回流到首页卡片，导致用户必须点进去才知道“为什么值得跟”。

详情页应继续承担完整转化，但进入体验必须稳定从顶部开始，并强化“先信任团长，再确认商品，再下单”的顺序。

### 2.3 团长主页

团长主页已经具备可信主页基础：

- 团长/店铺名称；
- 店铺简介；
- 成员数和订阅数；
- 订阅按钮；
- 履约方式；
- 当前团购；
- 店铺位置。

主要问题是：

- 团购列表与首页卡片重复度高，缺少“这是某个团长的经营主页”的独特表达；
- 顶部信任区还可以更集中地表达团长人设、经营范围和复购关系；
- 当前团购列表应体现“这个团长正在开的活动”，而不是像普通货架列表。

团长主页应更像“可信经营主页”，不是传统店铺货架，也不是首页列表的简单子集。

### 2.4 开团页

开团页作为经营入口完整度较好，能表达：

- 团长经营入口；
- 微信群需求整理成团购；
- 普通团购当前可用；
- 预售、卡券、报名暂未开放；
- 开团前准备事项。

主要问题不在开团页本身，而在首页过早、过重地承担了开团教育。

首页只需要保留轻入口；完整的开团理念、方式和准备事项由 `/open-group` 承接。

### 2.5 登录承接

登录页文案能说明登录后查看订单、订阅团长、继续跟团购买。

后续可以按来源增强承接感：

- 从“跟团购买”进入登录时，强调登录后回到当前团购继续下单；
- 从“订阅团长”进入登录时，强调登录后继续订阅；
- 从“开团”进入登录时，强调登录后创建店铺并发布团购。

### 2.6 桌面宽度访问

公开 H5 页面在桌面宽度下被横向拉满，视觉密度松散，不符合微信 H5 感。

后续应让 H5 公开页面在宽屏下保持移动端内容宽度约束，避免首页卡片、底部 Tabbar、搜索和团购列表铺满桌面。

## 3. 页面优化目标

本轮页面优化应达到以下目标：

- 首页移动端首屏能完整看到至少一张团购活动卡；
- 用户 3 秒内能从首页判断：谁在开团、卖什么、多少钱、何时结束或履约、多少人已跟；
- 团购详情首屏能形成信任与购买闭环：团长可信、活动清楚、商品明确、主操作突出；
- 团长主页看起来是团长可信经营主页，而不是普通店铺货架；
- 开团入口存在但不抢买家首页主线；
- H5 页面在桌面宽度下仍保持移动端页面观感；
- 所有优化保持“活动 + 团长 + 履约”表达，不转向低价商城、搜索商城或品牌官网。

## 4. 前端落实计划

### 4.1 首页信息层级重排

落点：`IndexView.vue`。

计划：

- 将 `index-hero` 压缩为轻量顶部区域：
  - 保留品牌、私域团购语境和一个弱化的一键开团入口；
  - 将平台统计改为一行轻量摘要，如“3 个进行中团购 · 162 人已跟团 · 1 位团长在经营”；
  - 移除或压缩首屏三步流程卡，不再占据独立大区块。
- 将“正在团购”作为首页主内容起点：
  - 标题下方直接进入活动卡；
  - 搜索、分类、定位改为辅助浏览区，避免挡在第一张活动卡之前。
- 定位提示改为轻量条：
  - 未开启定位时占用较小高度；
  - 点击附近筛选时再强化定位动作；
  - 关闭后不再影响首屏活动露出。
- 保持底部 Tabbar 不变，但首页内的开团入口只保留一个轻量入口。

验收：

- 390x844 移动视口首屏能看到完整首张团购卡或至少看到首张卡的主操作与价格；
- 360px 小屏不出现文字重叠、按钮挤压或横向滚动；
- 1280px 桌面宽度下 H5 内容不横向铺满。

### 4.2 团购卡片升级

落点：`GroupBuyFeedCard.vue`，必要时补充一个轻量展示工具函数。

计划：

- 保留团长行：团长头像、团长名、店铺名、订阅按钮。
- 将泛化履约标签升级为具体活动信号：
  - 状态：正在开团 / 已结束；
  - 时间：今晚截单、N 小时后结束、N 天后结束；
  - 参与：N 人已团；
  - 履约：配送/自提/附近可履约/距离信息。
- 增加一条活动理由文案：
  - 优先使用团购介绍的短句；
  - 无介绍时使用“买的是商品，也是团长的服务承诺”作为兜底；
  - 文案限制为一到两行，避免卡片变长。
- 优化图片 fallback：
  - 有真实图时继续展示真实商品图；
  - 无图时使用更可信的商品类 fallback，避免大面积文字色块拉低质感。
- 控制卡片按钮数量：
  - 主操作保持“去跟团”；
  - 分享和订阅降为次级，不抢购买主线。

验收：

- 卡片不是普通商品卡，必须同时出现团长/店铺、活动标题、价格、状态和主操作；
- 同一列表多张卡不再只重复同一组泛化标签；
- 无图卡片仍保持可信、不突兀。

### 4.3 团购详情页动线优化

落点：`GroupBuyDetailView.vue` 和现有导航/滚动恢复逻辑。

计划：

- 修正进入详情滚动行为：
  - 新进入 `/group-buys/:id` 默认滚到详情顶部；
  - 从详情返回列表时仍恢复列表滚动位置。
- 保持详情页信息顺序：
  - 团长/店铺信任；
  - 活动标题、价格、销量、库存、结束时间；
  - 热销商品；
  - 活动介绍；
  - 商品列表；
  - 履约说明；
  - 底部购买栏。
- 强化首屏可读性：
  - 顶部信任区和活动信息避免互相挤压；
  - 热销商品区明确它是“本团热销”，不让详情页退化为单商品页。
- 将详情页活动介绍中的短句回流给首页卡片。

验收：

- 直达详情、从首页点击详情、从团长主页点击详情，都从顶部开始；
- 首屏能回答谁在卖、卖什么、多少钱、还能不能买、什么时候结束或履约；
- 底部购买栏不遮挡最后一项内容。

### 4.4 团长主页优化

落点：`LeaderDetailView.vue`。

计划：

- 顶部信任区强化经营主页感：
  - 团长头像、名称、店铺名称、简介、成员/订阅数、订阅按钮集中展示；
  - 履约方式、当前团购、位置作为可信补充。
- 当前团购列表弱化重复店铺信息：
  - 在团长主页场景下可隐藏卡片重复的团长/店铺头；
  - 但保留活动标题、价格、状态、已团、履约信号和主操作。
- 保留轻排序：
  - 默认、上新、销量足够；
  - 不新增复杂筛选或商城频道。

验收：

- 页面首屏能明确这是某个团长的可信经营主页；
- 当前团购列表仍表现为活动列表，而非普通商品货架；
- 订阅入口清楚，但不压过正在团购活动。

### 4.5 登录与开团承接优化

落点：`LoginView.vue`、`OpenGroupView.vue`，必要时读取 redirect 来源。

计划：

- 登录页根据 redirect 提供更贴合来源的说明：
  - `/group-buys/:id`：登录后继续跟团购买；
  - `/leaders/:id`：登录后继续订阅团长；
  - `/open-group`：登录后创建店铺并发布团购；
  - 默认保持现有说明。
- 开团页继续承接完整经营说明：
  - 首页不再重复大段教育；
  - 开团页保留普通团购主入口和未开放类型灰态。

验收：

- 未登录点击跟团后进入登录页，用户能知道登录后会回到当前团购；
- 未登录点击开团后进入登录页，用户能知道登录后进入经营流程；
- 非当前可用能力不抢主流程。

### 4.6 宽屏 H5 约束

落点：`PageLayout.vue` 或公开 H5 页面外层样式。

计划：

- 对 H5 公开页面增加宽屏内容约束：
  - 移动端宽度不变；
  - 桌面宽度下内容保持约 430px 到 480px 的移动 H5 宽度，并居中展示；
  - 底部 Tabbar 与主内容宽度一致。
- 不影响 `/merchant/**` 管理端宽屏布局和横向滚动能力。

验收：

- 首页、团购详情、团长主页在 1280px 视口不横向铺满；
- 商家管理端不受该约束影响。

## 6. 明确不做

- 不把首页改成搜索商城首页；
- 不做低价补贴、秒杀、平台频道、复杂推荐算法；
- 不新增后端接口或数据模型；
- 不改订单、支付、订阅、优惠券等业务规则；
- 不新增真实地图 SDK；
- 不扩展未开放的卡券团购、报名团购等能力。
