# 联调文档维护规则

> 每个 batch 完成后必须同步更新 `docs/前后端联调文档.md`，不等待所有后端接口完成后再统一联调。

## 1. 更新范围

每个功能链路至少维护以下内容：

| 项目 | 要求 |
|---|---|
| 页面入口 | 说明来自哪个页面或操作 |
| 前置条件 | 登录态、数据准备、状态要求 |
| 调用顺序 | 按页面真实调用顺序写接口 |
| 请求样例 | 可直接用于联调 |
| 响应样例 | 字段必须与实际 DTO 一致 |
| 成功判定 | 包含页面状态、数据状态和业务状态 |
| 常见错误码 | 覆盖本批已实现异常路径 |
| 测试数据准备方式 | 能从空库按步骤准备 |

## 2. Batch 更新要求

| Batch | 更新重点 |
|---|---|
| Batch 0 | 通用响应、认证请求头、错误响应、本地联调基础约定、空库初始化 |
| Batch 1 | 模拟登录与获取当前用户 |
| Batch 2 | 创建店铺 |
| Batch 3A | 商品创建和选择说明 |
| Batch 3B | 发布普通团购，内联创建商品和复用已有商品 |
| Batch 3C | 我的团购列表、详情、更新、结束 |
| Batch 4 | 公共浏览，注明 Batch 9 前 `viewer.subscribed=false` |
| Batch 5 | 地址管理与订单预览 |
| Batch 6 | 创建订单、订单详情、取消订单 |
| Batch 7 | 模拟支付、库存变化、会员关系变化 |
| Batch 8 | 团长发货 |
| Batch 9 | 订阅团长、会员卡展示，并回填公共浏览真实订阅状态 |
| Batch 10 | 冻结 MVP 联调说明、测试账号、测试数据 |

## 3. 文档一致性

如果联调时发现 API 文档、数据模型、页面交互或实现存在差异，必须先明确差异原因，再决定修正文档或实现。不得私自修改 API 字段、状态枚举、错误码或金额口径。
