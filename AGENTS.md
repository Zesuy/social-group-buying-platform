# AGENTS.md

> 定位：本文是 AI 开发本项目时必须遵守的总规则。具体 batch 执行文件位于 `docs/dev/batches/`。

## 1. 项目目标

本项目实现一个以微信私域传播为主要场景的社群团购 MVP。核心闭环是：用户浏览团购、下单、模拟支付、团长发货；普通用户创建店铺后成为团长，并发布普通团购。

## 2. MVP 范围

MVP 只实现用户、团长、店铺、商品、普通团购、地址、订单、模拟支付、发货、订阅和基础会员卡展示。

必须遵循以下文档：

| 文档 | 用途 |
|---|---|
| `docs/功能需求定义.md` | 功能范围和优先级 |
| `docs/数据模型设计.md` | 数据对象、表结构和业务约束 |
| `docs/API风格规范.md` | API 风格、响应结构、错误码、金额和状态口径 |
| `docs/API设计.md` | MVP API 端点、请求响应和业务规则 |
| `docs/页面与交互文档.md` | 页面入口和交互流程 |
| `docs/前后端联调文档.md` | 前后端联调链路和样例 |
| `docs/dev/batches/` | 单 batch 执行文件 |

## 3. 禁止扩展项

不得实现以下非 MVP 能力：

| 能力 | MVP 处理方式 |
|---|---|
| 真实微信支付 | 仅使用模拟支付 |
| 优惠券 / 红包 | 不参与订单金额 |
| 售后退款完整流程 | 只预留状态，不实现申请、审核、退款 |
| 购物车 | 直接购买 |
| 帮卖分销 | 不实现帮卖关系、佣金和结算 |
| 积分商城 | 不建立积分账户、积分流水和兑换 |
| 公众号推送 | 不实现真实触达 |
| 平台后台 | 不建设后台页面和后台 API |

## 4. 技术栈

后端：Java 17 + Spring Boot + Maven + Spring Web + Validation + MyBatis-Plus + MySQL + Flyway + JUnit 5 + MockMvc。

前端：H5 移动端网页，Vue 3 + Vite + TypeScript + Vue Router + Pinia + Vant + Axios，后续使用 Vitest / Playwright 测试。

## 5. 开发策略

| 规则 | 要求 |
|---|---|
| 当前 batch | 每次只能执行当前 batch |
| 执行顺序 | 不得跳 batch，不得提前实现后续 batch 接口 |
| 代码组织 | 后端代码按模块组织，如 auth、user、store、product、groupbuy、address、order、subscription、member |
| 批次组织 | 开发批次按功能链路推进，不按模块或 Controller 文件拆批次 |
| 联调节奏 | 每完成一个功能链路，必须同步更新 `docs/前后端联调文档.md` |
| 文档约束 | 不得绕过既有 API 文档、数据模型、页面与交互文档 |
| 问题处理 | 不得把已知问题留到下一批 |

## 6. 测试硬性要求

| 范围 | 要求 |
|---|---|
| 所有接口 | 每个接口至少有 MockMvc 测试 |
| 关键 Service | 必须有单元测试 |
| 订单 / 支付 / 库存 / 发货 | 必须有事务和异常路径测试 |
| 权限 | 覆盖未登录、非资源所有者、非团长身份、跨店铺访问 |
| 状态 | 覆盖订单、支付、发货、团购等状态流转 |
| 库存 | 覆盖库存不足和支付扣减 |
| 快照 | 覆盖订单商品快照和地址快照 |
| 重复动作 | 覆盖重复支付、重复发货 |
| 前端批次 | 每个前端 batch 必须运行 `npm run typecheck`、`npm run lint`、`npm run test:unit`、`npm run build` 和 `npm run test:e2e` |
| 前端 E2E | 每个前端 batch 必须补充或维护与本批页面 / 链路匹配的 Playwright E2E；首次运行前执行 `npx playwright install`；不得为了通过 E2E 绕过真实接口、登录拦截、状态流转或非 MVP 灰态约束 |

测试不得留到下一批。

## 7. 文档维护要求

每个 batch 完成后必须同步维护 `docs/前后端联调文档.md`，至少包含页面入口、前置条件、调用顺序、请求样例、响应样例、成功判定、常见错误码和测试数据准备方式。

如果实现中需要修改上游文档，必须明确说明原因，并优先保持 `docs/功能需求定义.md`、`docs/数据模型设计.md`、`docs/API风格规范.md`、`docs/API设计.md`、`docs/页面与交互文档.md` 的口径一致。

## 8. 每批完成后的输出

每个 batch 完成后必须输出：

| 项目 | 要求 |
|---|---|
| 已实现接口清单 | 列出本批完成的接口 |
| 已新增 / 修改文件清单 | 列出代码和配置文件 |
| 已新增 / 修改 Flyway 迁移清单 | 如无新增迁移，写“无” |
| 已新增 / 修改测试清单 | 列出 MockMvc 和 Service 测试 |
| 已更新联调文档的位置 | 指明章节或标题 |
| 测试运行命令和结果 | 写明命令和通过 / 失败；前端 batch 必须包含 `npm run test:e2e` 结果 |
| 本批未解决问题 | 如无则写“无” |
