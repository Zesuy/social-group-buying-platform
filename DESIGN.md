# DESIGN.md

> 定位：本文是本项目 H5 前端的视觉系统与组件规范。所有前端页面、组件和样式改动都必须遵守本文。
>
> 上游业务准则见 `docs/前端产品与页面设计准则.md`。本文负责回答“具体怎么设计、怎么落地”，不重新定义业务范围。

## 1. Design Direction

本项目不是传统低价商城，也不是平台货架电商。前端视觉应服务“私域社群里的低门槛开店 + 团购活动销售”。

整体气质：

- **可信**：团长、店铺、履约承诺要清楚可见。
- **轻量**：普通用户也能开店和发布团购，页面不能像复杂后台。
- **活动感**：团购是短周期销售活动，时间、库存、销量、发货说明要进入关键视野。
- **微信 H5 感**：移动端优先，接近微信内网页 / 小程序，而不是桌面商城。
- **真实成交感**：商品图、价格、状态、主操作清晰，不做空泛营销装饰。

禁止气质：

- 低价平台感；
- 大型商城感；
- ERP 后台感；
- 过度品牌官网感；
- 为了显得丰富而堆占位模块。

## 2. Color System

颜色必须通过语义角色使用，不允许页面内随机写一套新颜色。

| Token | Color | 用途 |
|---|---:|---|
| `--color-primary` | `#07c160` | 主操作、订阅成功、发布成功、当前状态 |
| `--color-primary-dark` | `#08964b` | 主按钮 pressed / active |
| `--color-primary-soft` | `#e8f8ef` | 轻量主色背景、团长信任区、成功提示底色 |
| `--color-price` | `#ee3f2f` | 活动价、支付金额、成交关键数字 |
| `--color-price-soft` | `#fff1ee` | 价格标签、优惠感弱底色 |
| `--color-warning` | `#ff9f18` | 倒计时、库存紧张、待处理状态 |
| `--color-warning-soft` | `#fff6e7` | 待处理状态底色 |
| `--color-danger` | `#d93026` | 取消、失败、错误、不可逆操作 |
| `--color-page` | `#f6f7f8` | 页面背景 |
| `--color-surface` | `#ffffff` | 卡片、表单、弹层主体 |
| `--color-surface-muted` | `#f2f4f5` | 次级区块、输入背景、轻量分隔 |
| `--color-text` | `#1f2329` | 主文本、标题 |
| `--color-text-secondary` | `#646a73` | 描述、辅助信息 |
| `--color-text-muted` | `#8f959e` | 弱提示、时间、空态说明 |
| `--color-border` | `#e5e6eb` | 分割线、描边、表单边界 |

使用规则：

- 主色只服务可行动作和正向状态，不用于大面积装饰。
- 价格色只服务交易金额，不用于普通按钮和标题。
- 警告色用于时间、库存、待处理，不得和价格混用。
- 页面背景保持浅灰，内容区以白色实体卡片为主。
- 深色块只允许用于少量强调卡，不做大面积黑底页面。

## 3. Typography

字体使用系统栈：

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
  "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
```

字号层级：

| Token | Size | Weight | Line height | 用途 |
|---|---:|---:|---:|---|
| `--font-title-xl` | `24px` | 700 | `32px` | 关键页面标题、金额大数字 |
| `--font-title-lg` | `20px` | 700 | `28px` | 团购详情标题、店铺名 |
| `--font-title` | `18px` | 600 | `26px` | 区块标题、卡片主标题 |
| `--font-body` | `16px` | 400 | `24px` | 正文、表单输入 |
| `--font-body-sm` | `14px` | 400 | `22px` | 卡片描述、状态说明 |
| `--font-caption` | `12px` | 400 | `18px` | 标签、弱提示、时间 |
| `--font-price` | `22px` | 700 | `28px` | 活动价 |

规则：

- 移动端正文不得小于 `14px`；关键说明和按钮建议 `16px`。
- 价格使用 tabular numbers，避免金额跳动。
- 标题通过字重和间距建立层级，不使用过多颜色。
- 辅助文字可弱化，但不能低对比到看不清。
- 不使用负 letter-spacing。

## 4. Spacing And Layout

使用 4px 基础栅格，结构间距优先从以下值选择：

```text
4 / 8 / 12 / 16 / 20 / 24 / 32
```

常用规则：

| 场景 | 值 |
|---|---:|
| 页面左右边距 | `12px` 或 `16px` |
| 卡片内边距 | `12px` 或 `16px` |
| 同组信息间距 | `4px` 或 `8px` |
| 不同区块间距 | `12px` 或 `16px` |
| 页面顶部内容间距 | `12px` |
| 底部固定栏高度 | `56px` 起，叠加 safe-area |
| 可点击区域最小高度 | `44px` |

布局规则：

- 移动端优先按 375px 宽度设计，同时兼容 320px 小屏和 768px 宽屏。
- 页面内容不得出现横向滚动。
- 固定底栏、Tabbar、购买栏必须给页面内容预留底部 padding。
- 列表页只保留主滚动区，不做复杂嵌套滚动。
- 区块标题直接落在页面背景或内容流中，避免“大卡套小卡”。

## 5. Radius, Border And Shadow

| Token | Value | 用途 |
|---|---:|---|
| `--radius-xs` | `4px` | 小标签、输入内部元素 |
| `--radius-sm` | `8px` | 小按钮、状态标签 |
| `--radius-md` | `12px` | 卡片、表单分组、弹层 |
| `--radius-lg` | `16px` | 重点主卡、店铺头图区 |
| `--radius-pill` | `999px` | 胶囊按钮、状态 chip |

阴影：

```css
--shadow-card: 0 2px 10px rgba(31, 35, 41, 0.06);
--shadow-bottom-bar: 0 -4px 12px rgba(31, 35, 41, 0.08);
```

规则：

- 普通卡片优先用白底 + 轻阴影 / 细边框，不使用厚重投影。
- 不使用彩色发光、霓虹、过度玻璃拟态。
- 同一个页面只允许一种主体卡片圆角语言。
- 禁止卡片里再套同类卡片造成层级混乱。

## 6. Core Components

### 6.1 Primary Button

用于页面最重要的动作，如 `跟团购买`、`发布团购`、`一键开店`、`模拟支付`、`发货`。

样式：

- 高度：`44px` 或 `48px`；
- 圆角：`999px`；
- 背景：`--color-primary`；
- 文本：白色，`16px / 600`；
- pressed：背景使用 `--color-primary-dark`，或 `opacity: 0.9`；
- loading 时禁用重复点击。

规则：

- 每个页面最多一个最突出的主按钮。
- 主按钮文案必须是业务动作，不写“确定”“提交”这种泛词，除非上下文非常明确。

### 6.2 Secondary Button

用于次要动作，如 `查看详情`、`取消订阅`、`编辑地址`。

样式：

- 高度：`36px` 或 `40px`；
- 白底或浅灰底；
- 边框：`--color-border`；
- 文本：`--color-text` 或 `--color-primary`；
- 圆角：`999px` 或 `8px`。

### 6.3 Dangerous Button

用于取消订单、删除、退出等危险动作。

规则：

- 不得和主按钮并列同等强调。
- 使用 `--color-danger`，并与普通操作保持空间分离。
- 必须有确认或明确状态约束。

### 6.4 Group Buy Card

团购卡片不是普通商品卡，必须表达“活动 + 团长 + 商品 + 状态”。

必须包含：

- 团长头像 / 团长名 / 店铺名；
- 商品图；
- 团购标题；
- 活动价；
- 至少一个活动状态：销量、库存、结束时间、发货时间；
- 主操作：`去跟团` 或 `查看团购`。

推荐结构：

```text
团长行
商品图 + 活动信息
价格 / 状态 / 主操作
```

禁止：

- 只有商品图、标题、价格；
- 只突出低价，不展示团长；
- 卡片内按钮过多；
- 使用无业务意义的标签撑满视觉。

### 6.5 Leader Trust Block

用于首页卡片、团购详情页、团长主页。

应包含：

- 团长头像；
- 团长名称；
- 店铺名称；
- 简短简介或履约说明；
- 订阅状态 / 订阅按钮；
- 当前开团、复购、发货等基础信任信息。

样式：

- 白底或 `--color-primary-soft`；
- 圆角 `12px`；
- 内边距 `12px`；
- 头像尺寸 `40px` 或 `48px`。

### 6.6 Status Chip

用于订单状态、团购状态、库存状态。

| 状态 | 颜色 |
|---|---|
| 进行中 / 已发布 / 已支付 | 主色 |
| 待支付 / 待发货 / 库存紧张 | 警告色 |
| 已取消 / 失败 / 不可购买 | 危险色或灰色 |
| 已完成 / 已发货 | 中性色或主色弱化 |

规则：

- 状态不能只靠颜色表达，必须有文字。
- 状态 chip 不应大面积占据页面。

### 6.7 Bottom Action Bar

用于详情页购买、订单操作、发布保存。

样式：

- 固定底部；
- 高度 `56px` 起；
- 白底；
- 顶部轻阴影；
- padding-bottom 包含 `env(safe-area-inset-bottom)`；
- 主按钮靠右或全宽，次要操作降级。

规则：

- 页面内容必须预留底部空间。
- 不得遮挡列表最后一项、表单错误、价格明细。

### 6.8 Forms

用于创建店铺、发布团购、地址、发货。

规则：

- 表单按业务分组，不做一个长列表到底。
- label 必须可见，不只依赖 placeholder。
- 错误提示放在字段附近。
- 提交按钮 loading + disabled。
- 发布团购表单必须按 `介绍 -> 商品 -> 设置 -> 发布` 组织。

## 7. Page Patterns

### 7.1 Home

目标：公开团购活动信息流。

首屏优先级：

1. 平台身份和私域团购语境；
2. 正在进行的团购活动；
3. 团长 / 店铺可信信息；
4. 一键开团入口；
5. 订单和个人中心入口。

不得做成搜索商城首页或纯商品瀑布流。

### 7.2 Group Buy Detail

目标：完成浏览到下单的转化。

信息顺序：

1. 商品封面 / 活动主图；
2. 标题、活动价、销量、库存、结束时间；
3. 团长 / 店铺信任区；
4. 发货说明；
5. 商品和活动介绍；
6. 底部购买栏。

首屏必须让用户知道谁在卖、卖什么、多少钱、还能不能买、怎么下单。

### 7.3 Leader Home

目标：建立团长可信主页。

必须突出：

- 团长身份；
- 店铺信息；
- 订阅入口；
- 当前团购活动；
- 可复购关系。

不得做成传统店铺货架。

### 7.4 Publish Group Buy

目标：帮助轻商家创建一次销售活动。

结构必须是：

```text
介绍 -> 商品 -> 设置 -> 发布
```

不得做成后台式字段堆叠。复杂设置应折叠或分组。

### 7.5 Orders

买家订单突出订单状态和下一步动作。

团长订单突出待发货、团购活动归属、收货信息和防重复发货。

## 8. Vant Usage

Vant 是基础组件库，不是最终设计系统。

必须二次处理：

- 页面间距；
- 信息层级；
- 主次按钮；
- 卡片形态；
- 状态标签；
- 空状态；
- loading / disabled；
- safe-area；
- 错误提示。

优先使用 Vant：

- 表单；
- Dialog / Toast；
- Tabs；
- PullRefresh / List；
- Empty；
- Skeleton；
- Tabbar；
- SwipeCell；
- Stepper；
- Picker / Popup。

不得因为 Vant 默认样式可用，就忽略业务信息层级。

## 9. Empty, Loading And Error

所有列表和异步页面必须包含：

- loading：骨架屏优先，超过 300ms 不应空白；
- empty：说明当前为空的原因，并给出可用动作；
- error：说明失败原因，并提供重试；
- disabled：提交中不可重复点击；
- success：关键动作完成后给出反馈。

空态文案必须服务业务，不写泛泛的“暂无数据”作为唯一说明。

示例：

| 场景 | 文案方向 |
|---|---|
| 首页无团购 | 暂无正在进行的团购，可去一键开团 |
| 未创建店铺 | 开店后即可发布自己的团购活动 |
| 无订单 | 跟团购买后，订单会出现在这里 |
| 无订阅 | 订阅团长后，可更快找到后续团购 |

## 10. Non-MVP And Placeholder Rules

禁止无意义占位组件。

非 MVP 功能可以隐藏或灰态，但必须：

- 不调用不存在的接口；
- 不误导用户以为当前可用；
- 点击后有明确提示；
- 不抢占主流程位置。

非 MVP 能力包括：

- 真实微信支付；
- 优惠券 / 红包；
- 完整售后退款；
- 购物车；
- 帮卖分销；
- 积分商城；
- 公众号真实推送；
- 平台后台。

## 11. Motion

动效只服务状态变化和触控反馈。

规则：

- 微交互时长 `150ms-300ms`；
- 使用 opacity / transform；
- 不动画 width / height；
- 尊重 `prefers-reduced-motion`；
- 列表加载可轻微淡入，但不做夸张入场；
- 按钮点击必须有反馈，但不能导致布局跳动。

## 12. Accessibility

最低要求：

- 交互区域不小于 `44px`；
- 图标按钮必须有可访问名称；
- 表单字段有 label；
- 错误不只靠颜色表达；
- 文本对比度满足可读性；
- 页面可缩放，不禁用用户缩放；
- 固定底栏不遮挡系统手势区域。

## 13. UI Review Checklist

每次前端 UI 改动完成后必须自检：

- 页面是否表达“私域团购活动”，而不是普通商城；
- 首屏是否能看见团购、团长、价格、状态或主操作；
- 团购卡片是否包含团长 / 店铺信息；
- 详情页是否说明结束时间、库存或发货承诺；
- 发布页是否按 `介绍 -> 商品 -> 设置 -> 发布` 组织；
- 是否存在无意义占位组件；
- 每页是否只有一个最突出的主操作；
- 固定底栏是否遮挡内容；
- loading、empty、error、disabled 是否完整；
- 是否误强化了非 MVP 功能；
- 是否运行了本 batch 要求的前端测试命令。
